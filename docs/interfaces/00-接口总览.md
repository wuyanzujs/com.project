# 接口总览

这套文档按业务模块拆分，每个接口单独一个 Markdown 文件，方便在 Obsidian 里从左侧目录或双链进入。

## 请求公共规则

- 业务接口主要通过 `depponHttp` 调用。
- 基础域名来自 `APP_RUNTIME_CONFIG.apiBaseURL`。
- `depponHttp` 会自动携带 `ECO_TOKEN` cookie。
- 后端响应会归一成 `DepponResponse<TResult>`。
- 文档里的“请求字段”展开的是业务请求体或 URL/query 参数；发票模块会额外说明公共 `path` 包装。
- 字段说明根据源码类型、字段命名、service 调用语义整理；打印模块里尚未真正封装的接口会明确标为“待接入”。
