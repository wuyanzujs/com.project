# 按运单号查询可开票订单

返回 [[09-发票/00-模块总览|发票接口]]，也可以回到 [[00-接口总览]]。

## 接口说明

| 项目      | 说明                                                                  |
| --------- | --------------------------------------------------------------------- |
| 用途      | 按运单号查询可开票订单。                                              |
| 源码方法  | `invoiceOrderSearchService.queryByWaybill`                            |
| 源码位置  | `apps/com.deppon.app/src/services/invoice/orderSearch.service.ts:99`  |
| HTTP 方法 | `POST`                                                                |
| 发票 path | `tradeQueryBySourceBillNo`                                            |
| 接口路径  | `/gwapi/onlineService/eco/online/invoice/secure/invoiceCommonService` |
| 登录态    | 是                                                                    |
| 请求类型  | `InvoiceOrderQueryRequest`                                            |
| 响应类型  | `InvoiceOrderListResponse`                                            |

## 调用说明

这个接口走发票公共网关，真实 HTTP path 固定为 `/gwapi/onlineService/eco/online/invoice/secure/invoiceCommonService`，业务动作由请求体里的 `path` 区分。字段表只展开 `data` 部分，公共外层结构是 `{ path, data }`。

## 请求字段

类型：`InvoiceOrderQueryRequest`。

| 字段           | 必填 | 类型     | 说明                                 |
| -------------- | ---- | -------- | ------------------------------------ |
| `sourceBillNo` | 是   | `string` | 来源单号，当前开票场景通常是运单号。 |

## 响应字段

类型：`InvoiceOrderListResponse`。

| 字段        | 必填 | 类型                | 说明                                  |
| ----------- | ---- | ------------------- | ------------------------------------- |
| `total`     | 是   | `number`            | 总记录数。                            |
| `pageSize`  | 是   | `number`            | 每页条数。                            |
| `pageIndex` | 是   | `number`            | 页码，从当前业务传参看通常从 1 开始。 |
| `list`      | 是   | `InvoiceOrderRaw[]` | 列表数据。                            |
