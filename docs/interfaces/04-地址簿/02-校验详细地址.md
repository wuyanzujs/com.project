# 校验详细地址

返回 [[04-地址簿/00-模块总览|地址簿接口]]，也可以回到 [[00-接口总览]]。

## 接口说明

| 项目      | 说明                                                              |
| --------- | ----------------------------------------------------------------- |
| 用途      | 校验地址是否足够详细。                                            |
| 源码方法  | `contactApi.checkAddressDetail`                                   |
| 源码位置  | `apps/com.deppon.app/src/services/contact/contact.api.ts:121`     |
| HTTP 方法 | `POST`                                                            |
| 接口路径  | `/gwapi/userService/eco/user/contact/secure/checkAddressIsDetail` |
| 登录态    | 默认是                                                            |
| 请求类型  | `ContactAddressCheckRequest`                                      |
| 响应类型  | 只消费统一 OWS `status/message`，`result` 当前不参与业务判断。     |

## 请求字段

类型：`ContactAddressCheckRequest`。

| 字段       | 必填 | 类型     | 说明       |
| ---------- | ---- | -------- | ---------- |
| `province` | 是   | `string` | 省份名称。 |
| `city`     | 是   | `string` | 城市名称。 |
| `county`   | 是   | `string` | 区县名称。 |
| `address`  | 是   | `string` | 详细地址。 |

## 响应与页面决策

| 条件 | 领域结果 | 页面行为 |
| --- | --- | --- |
| `status=true` | `pass` | 继续选择或保存。 |
| `status=false` 且有业务 `message` | `review` | 展示后端提示，让用户选择修改地址或继续使用。 |
| `status=false` 且无业务 `message` | `review` | 使用通用提示，不能静默当作校验成功。 |
| `transportFailure=true` | `unavailable` | 提示校验暂不可用，但不阻断地址选择或保存。 |
| `authExpired=true` | `blocked` | 中止当前动作，交给统一登录失效流程。 |

`pages/contact/list` 与 `pages/contact/edit` 统一通过 `useContactAddressIntegrity` 消费该决策；页面不能直接调用 `contactApi` 或自行解释响应。
